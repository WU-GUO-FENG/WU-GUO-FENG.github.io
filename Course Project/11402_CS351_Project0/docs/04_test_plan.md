# Test Plan (測試計畫) - Two Sum Project

## 1. 引言

### 1.1 目的

本文件定義「Project 0 - Two Sum」的測試策略、測試案例與驗證標準。透過單元測試與壓力測試，確保 TwoSumArray 與 TwoSumHashTable 兩個演算法在各種邊界條件下皆能正確運行，且符合複雜度預期。

### 1.2 測試範圍

- 功能性測試：驗證演算法在正常、負數、重複數字、無解等情境下的正確性。
- 性能測試：驗證哈希表法在大規模數據下的處理能力。
- 自動化測試：確保所有測試能在 GitHub Actions CI 環境與 Docker 容器中執行。

## 2. 測試環境與工具

- 測試框架：Google Test (GTest)
- 編譯器：GCC 13+ / Clang 16+ (支援 C++23)
- 建構系統：CMake
- 自動化工具：GitHub Actions & Docker

## 4. 測試執行流程

### 4.1 本地執行

進入 build 資料夾：

```
mkdir build && cd build
```

配置 CMake：

```
cmake ..
```

編譯專案：

```
make
```

執行測試：

```
./tests/test_two_sum
```

### 4.2 自動化 CI 執行

當程式碼推送到 GitHub 或發起 Pull Request 時：

- 環境觸發：GitHub Actions 啟動 Ubuntu Runner。
- Docker 建構：執行 docker build。
- 容器測試：Docker 內部執行編譯與 GTest 執行檔。
- 結果回報：若有任一 Test Case 失敗，CI 狀態顯示紅燈並阻斷 Merge。

## 5. 通過準則 (Pass/Fail Criteria)

**通過 (Pass)：** 所有 10 個測試案例均回傳預期數值，且無記憶體洩漏（可透過 Valgrind 輔助檢查）。

**失敗 (Fail)：** 任一測試案例回傳錯誤索引、程式崩潰（Crash）或執行時間超出預期。
